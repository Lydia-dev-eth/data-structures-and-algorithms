 # Data Structures and Algorithms

This repository documents my journey of learning Data Structures and Algorithms through the **MIT 6.006: Introduction to Algorithms** course, programming assignments, and algorithmic problem solving on LeetCode.

The goal of this repository is to build a strong foundation in computer science by understanding both the theory behind algorithms and their practical applications.

---

# Repository Structure

```text
data-structures-and-algorithms/
│
├── mit-6006/
├── leetcode/
├── psets/
└── README.md
```

---

## `mit-6006/`

This directory contains my work while following the MIT 6.006 course.

Each lecture has its own directory containing:

* **README.md** – Overview of the lecture, including the topics covered and the files included.
* **notes.md** – Personal notes summarizing the lecture in my own words.
* **Python files** – Implementations and experiments based on the lecture material.

Example:

```text
mit-6006/
└── lecture-02-dynamic-arrays/
    ├── README.md
    ├── notes.md
    └── dynamic_array.py
```

The purpose of this section is to understand how algorithms and data structures work by studying the lecture material and implementing the concepts from scratch.

---

## `leetcode/`

This directory contains my LeetCode solutions organized by topic rather than difficulty.

Example:

```text
leetcode/
├── arrays/
│   ├── 0026_remove_duplicates.py
│   └── 0088_merge_sorted_array.py
│
├── binary-search/
├── trees/
└── graphs/
```

Each solution includes:

* Problem number and title
* Problem link
* Solution
* Time complexity
* Space complexity
* Brief explanation of the approach

This section focuses on applying data structures and algorithms to interview-style problems and improving problem-solving skills.

---

## `psets/`

This directory contains programming assignments completed as part of university courses and online courses.

Unlike LeetCode problems, these assignments are larger projects that often involve multiple classes, algorithms, and files.

Example:

```text
psets/
└── mit/
    └── ps4/
        ├── README.md
        ├── ps4.py
        ├── ps4_tests.py
        └── notes.md
```

Each problem set typically includes:

* **README.md** – Assignment overview, objectives, and project structure.
* **Solution files** – Source code implementing the assignment.
* **Test files** – Unit tests or provided testing scripts.
* **notes.md** – Personal observations, challenges encountered, and lessons learned.

The purpose of this section is to document complete programming projects and demonstrate the practical application of algorithms and software engineering principles.

# Learning Workflow

For each new topic:

1. Study the concept through MIT 6.006.
2. Write personal notes summarizing the lecture.
3. Implement the concepts introduced in the lecture.
4. Solve related LeetCode problems to reinforce understanding.
5. Complete programming assignments that require applying multiple concepts together.
6. Review and improve previous work as my understanding grows.

# Repository Philosophy

This repository is more than a collection of solutions. It documents my learning process, emphasizing understanding, implementation, and continuous improvement.

Each section serves a different purpose:

* **MIT 6.006** provides the theoretical foundation.
* **LeetCode** provides focused algorithmic practice.
* **PSETs** provide experience applying multiple concepts to larger software projects.

Together, these components help build both a solid understanding of computer science fundamentals and practical programming skills.
